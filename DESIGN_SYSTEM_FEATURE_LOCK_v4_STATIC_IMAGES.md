# OTOS Design System Feature Lock v4 — Static Image Layers

User correction: all image layers should behave as static visual layers while all text/content scrolls.

Rules:
- Hero images are static visual backgrounds; hero copy scrolls normally.
- In-page image cards may use the same static-image treatment.
- Final route panes use a static image-backed glass field.
- Do not make whole content sections sticky.
- Do not reintroduce black slab backgrounds.
- Mobile may fall back to scroll backgrounds for Safari/device stability.
