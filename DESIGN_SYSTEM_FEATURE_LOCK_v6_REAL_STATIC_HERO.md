# OTOS Design System Feature Lock v6 — Real Static Hero Images

Canonical rule:
- Hero images must be real `<img class="hero-image">` elements, not background-only.
- On desktop, the hero image layer may be fixed/static while hero text and all content scroll normally.
- On mobile, hero images fall back to normal absolute positioning.
- The header remains fixed above all image layers.
- Content cards may use subtle OTOS brain watermarks.
- Final route panes use a transparent glass frame.
